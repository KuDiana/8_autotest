# API Autotests

Проект содержит набор автотестов для проверки API-запросов системы авторизации и профилей пользователей.

Используемые технологии:
- Python
- Pytest
- Requests
---

# Предварительная настройка

Перед запуском тестов необходимо:

1. Открыть файл `config.py`
2. Ввести:
   - логин администратора
   - пароль администратора

Пример:

```python
BASE_URL = "https://secby.ru"

ADMIN_USERNAME = "admin"
ADMIN_PASSWORD = "123456"
```

---

# Запуск тестов

## Запуск всех тестов
ключ -s для вывода print в консоль.

```bash
pytest -v -s
```

---

## Запуск проверки авторизации

```bash
pytest tests/test_auth.py -v -s
```

### test_auth.py

#### test_login_success
Проверка успешной авторизации пользователя:
- ввод корректного логина и пароля
- получение access token
- проверка наличия токена в ответе

---

#### test_send_token
Проверка отправки access token:
- отправка токена в заголовке Authorization
- получение профиля пользователя
- проверка соответствия profile id

---

#### test_invalid_user_login
Проверка авторизации незарегистрированного пользователя:
- отправка несуществующего логина и пароля
- проверка получения ошибки 401 Unauthorized

---

#### test_validation_error
Проверка ошибки валидации:
- отправка некорректного payload
- проверка получения ошибки 422 Validation Error


---

## Запуск проверки User Profile

```bash
pytest tests/test_profile.py -v -s
```
### test_profile.py

#### test_get_user_profile
Проверка получения профиля пользователя:
- отправка access token пользователя
- получение User Profile
- проверка username и email
- проверка всех полей профиля:
  - id
  - username
  - email
  - role_id
  - profile_id
  - is_active
  - created_at
  - updated_at
  - role
- проверка полей объекта role:
  - id
  - name
  - description
- вывод полной информации профиля пользователя
---

## Запуск проверки Admin Profile

```bash
pytest tests/test_admin.py -v -s
```
### test_admin.py

#### test_get_admin_profile
Проверка получения профиля администратора:
- отправка access token администратора
- получение Admin Profile
- проверка роли admin
- проверка всех полей профиля:
  - id
  - username
  - email
  - role_id
  - profile_id
  - is_active
  - created_at
  - updated_at
  - role
- проверка полей объекта role:
  - id
  - name
  - description
- вывод полной информации администратора
---

#### test_admin_get_all_profiles
Проверка получения списка пользователей администратором:
- отправка access token администратора
- получение списка пользователей
- проверка успешного ответа
- вывод количества найденных пользователей
- вывод первых пользователей из списка

---

#### test_admin_change_user_role
Проверка назначения роли пользователю:
- изменение роли пользователя на admin
- получение обновлённого профиля
- проверка успешной смены роли
- вывод новой роли пользователя

---

#### test_admin_role_access
Проверка расширения прав после смены роли:
- назначение пользователю роли admin
- проверка доступа к списку пользователей
- проверка успешного получения данных
- вывод количества доступных пользователей

---

#### test_admin_delete_user
Проверка удаления пользователя администратором:
- создание нового пользователя
- получение profile id пользователя
- удаление пользователя администратором
- проверка успешного удаления
- защита от удаления администратора
- вывод информации об удалённом пользователе

# Особенности запуска

Во время выполнения тестов используется `print()` вывод.
Это позволяет визуально контролировать работу API.

---

# Архитектура проекта

## tests/

Содержит основные автотесты.

---

## utils/

### api_client.py

Содержит функции для отправки API-запросов:
- login
- register
- get_profile
- get_profiles
- change_role
- delete_user

---

## data/

### payload.py

Содержит:
- тестовые данные
- генерацию пользователей
- невалидные payload
- данные для проверки ошибок

---

## conftest.py
Содержит pytest fixtures:
- создание тестового пользователя
- предусловия
- постусловия
- автоматическое удаление тестовых пользователей
- получение admin token

# Реализованные автотесты

- ввод логина и пароля

### Проверка метода получения информации о пользователе

# Дополнительно реализовано

## Проверка ошибок API
- 401 Unauthorized
- 422 Validation Error

## Проверка ролевой модели
- назначение роли пользователю
- проверка расширения прав после смены роли

- получение списка пользователей
- удаление пользователя

- предусловия (регистрация пользователя)
- постусловия (удаление пользователя)
